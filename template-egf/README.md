# Template EGF

> **Status:** `DRAFT 0.1` — first draft for co-authoring by Founding Members.

📄 **[Read the Template EGF](./template-egf.md)**

A **scaffold, not a gate.** Any organization can author and operate an
Ecosystem Governance Framework on Verana with no Council approval, and using
this template implies no Council endorsement. The Template EGF is a working
scaffold the Council maintains to make authoring a sector EGF cheaper, safer,
and conformant with the [Network GF](../network-gf/) **by construction**.

The authoring guidance is integrated into the template itself (the
three-mode convention: suggested text, ⟨fill-in fields⟩, and *Instruction*
blocks to delete before publication) — there is no separate authoring guide.

## What it provides

- **Network GF interface checklist** — a conformance-by-construction annex
  mapping every mandatory EGF content item of the Network GF to the template
  section that satisfies it.
- **Per-schema configuration patterns** — the onboarding-mode menu (OPEN,
  `GRANTOR_ONBOARDING_PROCESS`, `ECOSYSTEM_ONBOARDING_PROCESS`), holder and
  verifier modes, validity periods, and pricing choices, each explained.
- **Lifecycle and sanctions boilerplate** — an eight-stage participant
  lifecycle and a graduated sanctions ladder pre-wired to the VPR's on-chain
  permission, revocation, and slashing semantics.
- **Publication instructions** — the exact VPR message sequence to publish a
  Corporation, an Ecosystem, the EGF documents, credential schemas, and root
  participants.

The [ECS-EGF](../ecs-egf/) is the worked example of this template.
